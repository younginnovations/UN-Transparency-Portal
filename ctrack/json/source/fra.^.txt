Tous les membres du OECD et un certain nombre d' autres pays et institutions rendent compte annuellement à la <span class="bold italic">OECD Système de notification des pays créanciers </span> sur l'aide publique au développement et d'autres flux officiels. En Décembre chaque année, le CRS est mis à jour avec les données de l'année précédente (si c'est entre 12 et 23 mois sur la date) Bien que les données passéiste n'est pas particulièrement utile pour la planification du développement - c'est un point de référence très utile et relativement précise. il est particulièrement utile pour inclure des espaces réservés pour les pays et les institutions qui ne relèvent pas actuellement de l'IATI. les données fournies dans cette colonne est le total des versements annuels (de dépenses) pour le donateurs. les valeurs sont en dollars courants des États-Unis. #data_title_text
Pourquoi les questions de la qualité des données

#data_paragraph1_text
La plupart des données disponibles actuellement dans d-portal vient des éditeurs de l'IATI : plus de 240 organisations qui se sont engagées à publier leurs activités de développement en format IATI-XML. Ces données sont publiées sur leurs propres sites Web et indexé sur le registre de l'IATI. Sur une base quotidienne sondages IATI toutes les données indexées sur le registre et enregistre la dernière version de chaque activité publié dans une banque de données centrale. Le magasin de données n'est donc pas une base de données organisée, mais une compilation de toutes les données de l'IATI comme de la nuit dernière.

#data_paragraph2_text
Le standard de l'IATI est flexible dans ses règles de déclaration pour permettre que de nombreuses organisations différentes avec une capacité variant de publier leurs données dans ce format commun. La philosophie est pour les organisations de commencer à publier ce qu'ils peuvent et améliorer la qualité de leurs données dans le temps. Cela signifie qu'il y aura toujours des données de qualité variable disponible par l'IATI.

#data_paragraph3_text
Cela dit que l'IATI a été créé pour répondre aux besoins des pays en développement, qui ont toujours demandé des informations en temps opportun, complet et prospective qui peuvent être utilisés dans la planification et le suivi du développement. <a href="http://www.aidtransparency.net/wp-content/uploads/2013/01/Paper-4a-Country-Survey-of-AIMS.pdf" target="_blank" title="Liens vers un fichier PDF dans une nouvelle fenêtre">Un récent sondage</a> menée par l'IATI renforcé ces exigences. IATI de poser est donc que les éditeurs mettent à jour leurs données au moins trimestriellement ; que les dernières données n'est pas plus de trois mois de retard ; et que les budgets au niveau de l'activité prospectives sont fournies pour au moins un an à l'avance.

#data_paragraph4_text
d-portal présente le meilleur de la pratique d'un certain nombre des principaux partenaires de développement. De nombreux éditeurs de l'IATI, cependant, sont actuellement <span class="underline">pas</span> réunion ce pays en développement ont clairement indiqué sont leurs exigences minimales. Ce n'est tout simplement pas assez bon.

#data_paragraph5_text
Lorsque nous avons commencé à développer <span class="bold italic">d-portal</span> nous pensions que notre tâche était de présenter les données de l'IATI. Nous nous sommes vite rendu compte que d'un point de vue sur la campagne ce fut de peu d'utilité : il est tout aussi important de montrer les données manquantes pour montrer où il est présent. Dans différents tableaux, vous verrez les cellules contenant "0" : ceux-ci indiquent les zones où les éditeurs de l'IATI ne parviennent pas à fournir des données. Vous verrez aussi les cellules contenant " - " : ceux-ci indiquent les organisations qui ne sont pas encore à publier des données de l'IATI.

#data_paragraph6_text
Nous espérons que notre présentation des données - et les lacunes - fournit un aperçu utile du paysage de données. Où il ya des trous dans les données critiques, nous vous encourageons à vous engager avec vos partenaires de développement d'exiger qu'ils améliorent la qualité des données qu'ils font actuellement disponible.
