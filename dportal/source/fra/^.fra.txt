#^about_paragraph_title_text
Qu'est-ce que c'est?

#^about_paragraph1_text
d-portal.org vous aide à explorer les données IATI en fournissant des informations par pays ou par l'éditeur sur les activités de développement et les budgets publiés à IATI.

#^about_paragraph2_text
Il vise à fournir des ministères de tutelle , les parlementaires et la société civile des informations qui peuvent aider à la planification et au suivi des activités de développement.

#^about_paragraph3_text
Dans sa première itération , il contient des données actuelles publiées par l'Initiative internationale pour la transparence de l'aide ( IATI ), ainsi que les plus récentes (2012) données publiées par la <a href="http://www.oecd.org/dac/stats/ crsguide.htm" target="_blank" class="co"> créanciers du CAD de l'OCDE système de notification </a>.

#^about_paragraph4_text
La plate-forme actuelle pose deux questions : d'où viennent les ressources proviennent , et où vont-ils ( par secteur et localisation) ? Des données récentes , en cours et à venir est présenté et comparé différentes sources de données.

#^about_paragraph5_text
Afin de répondre à ces questions dans une perspective de pays, il ne suffit pas simplement d'afficher des données disponibles :

#^about_paragraph6_text
est une plate-forme open source. Cela signifie que vous pouvez utiliser tous les aspects de la plate-forme pour construire votre propre portail en utilisant le code source libre.

#^about_paragraph7_text
Bien qu'il fournit actuellement un déploiement générique pour n'importe quel pays, il a été développé avec l'intention spécifique d'être personnalisée dans le pays pour répondre aux besoins locaux. Pour plus d'informations sur la façon dont vous pouvez créer une version personnalisée pour un pays spécifique, voir <a href="#install" class="how"> Comment installer </a>.

#^faq_general_q1_text
Qu’est-ce que le d-portal?

#^faq_general_a1_text
Il s’agit d’un outil donnant un aperçu de toutes les données de l’IITA (<a href="http://www.aidtransparency.net">Initiative internationale pour la transparence de l’aide</a>) par pays bénéficiaire ou organisation déclarante, comparable aux données récentes du SNPC  (<a href="https://stats.oecd.org/Index.aspx?DataSetCode=CRS1">Système de notification des pays créanciers</a>) du Comité d’aide au développement (CAD) de l’Organisation de coopération et de développement (OCDE).

#^faq_general_q2_text
Quels ensembles de données retrouve-t-on dans le d-portal?

#^faq_general_a2_text
Les données de l’<a href="http://www.aidtransparency.net">IITA</a> et du <a href="https://stats.oecd.org/Index.aspx?DataSetCode=CRS1">SNPC</a> (voir ci-dessous pour obtenir de plus amples renseignements).

#^faq_general_q3_text
Qu’est-ce que l’IITA?

#^faq_general_a3_text
L’<a href="http://www.aidtransparency.net">Initiative internationale pour la transparence de l’aide</a> (IITA) est une initiative volontaire et pluri-partite qui vise à améliorer la transparence de l’aide, du développement et des ressources humanitaires afin d’accroître l’efficacité de la lutte contre la pauvreté.

#^faq_general_q4_text
Qu’est-ce que le SNPC du CAD de l’OCDE?

#^faq_general_a4_text
Il s’agit du <a href="https://stats.oecd.org/Index.aspx?DataSetCode=CRS1">Système de notification des pays créanciers</a> du Comité d’aide au développement (CAD) de l’Organisation de coopération et de développement économiques (OCDE) qui contient des données sur l’aide publique au développement.

#^faq_general_q5_text
D’où proviennent les données?

#^faq_general_a5_text
Les données de l’IITA sont tirées du <a href="http://www.iatiregistry.org/">Registre de l’IITA</a>.
<p>Les données du SNPC proviennent des <a href="https://stats.oecd.org/Index.aspx?DataSetCode=CRS1">tableaux du SNPC</a>.</p>
<p>D’autres données sont consignées dans une feuille de calcul <a href="https://docs.google.com/spreadsheets/d/1jpXHDNmJ1WPdrkidEle0Ig13zLlXw4eV6WkbSy6kWk4/edit#gid=15">ctrack</a> de Google.</p>

#^faq_general_q6_text
Pourquoi manque-t-il des données à certains endroits dans le d-portal?

#^faq_general_a6_text
Les organisations présentent leurs données de l’IITA selon différents niveaux de détails. Elles les mettent à jour et les améliorent constamment. Si une organisation ne publie des données que pour 2013, elles n’apparaîtront pas dans les colonnes ou les graphiques de 2014.
<p>Le d-portal reflète simplement les données; aucune correction n’est apportée aux données ou aux informations publiées dans le cadre de l’IITA, sauf quelques exceptions pour rendre le site plus convivial.</p>
<p>Consultez aussi les informations sur la <a href="http://d-portal.org/ctrack.html?country=AF&tongue=eng#view=data_quality">qualité des données</a> (<i>l’Afghanistan est utilisé à titre d’exemple</i>)</p>

#^faq_general_q7_text
Pourquoi y a-t-il des zéros et des tirets dans les tableaux?

#^faq_general_a7_text
Les zéros peuvent indiquer un problème relatif à la qualité des données (p. ex. : codes, dates, etc. incompatible avec la norme de l’IITA). <i>Des liens vers le profil de l’organisation seront bientôt intégrés au <a href="http://dashboard.iatistandard.org/index.html">tableau de bord de l’IITA</a> afin d’examiner des problèmes de qualité des données précis.</i>
<p>Les tirets indiquent que le donateur ou l’organisme ne publie pas de données de l’IITA à l’heure actuelle.</p>

#^faq_general_q8_text
Quand et comment les données dans d-portal sont-elles mises à jour?

#^faq_general_a8_text
Les données de l’IITA sont mises à jour chaque nuit (en important les données qui ont été modifiées le jour précédent) et une importation complète des données a lieu chaque fin de semaine. Dans chaque cas, les données sont tirées du Registre de l’IITA.
<p>Les données du SNPC sont téléchargées annuellement et stockées dans <a href="https://docs.google.com/spreadsheets/d/1jpXHDNmJ1WPdrkidEle0Ig13zLlXw4eV6WkbSy6kWk4/edit#gid=15">ctrack</a>.</p>

Consultez le <a href="http://dashboard.iatistandard.org/index.html">tableau de bord de l’IITA</a>.

#^faq_general_q13_text
Quelles sont les raisons les plus courantes pour lesquelles les données d’une organisation n’apparaissent pas dans certaines sections du d-portal?

#^faq_general_a13_text
Données signalées comme xml de l’IITA invalide par le <a href="http://validator.iatistandard.org/">valideur</a>.
<p>Incohérence entre l’ « identifiant » dans le profil de l’organisation déclarante du <a href="https://iatiregistry.org/">Registre de l’IITA</a> et le <a href="http://iatistandard.org/202/activity-standard/iati-activities/iati-activity/reporting-org/">reporting-org</a> dans ses données – des zéros apparaîtront dans ses données jusqu’à ce que les deux correspondent.</p>
<p>Absence du <a href="http://iatistandard.org/202/codelists/TransactionType/">code du type de transaction</a>.</p>
<p>Absence de décaissement ou dépense dans le <a href="http://iatistandard.org/202/codelists/TransactionType/">type de transaction</a>.</p>
<p><a href="http://iatistandard.org/202/activity-standard/iati-activities/iati-activity/activity-date/">Dates</a> saisies dans un format incorrect.</p>
<p>Dates incohérentes, comme la date de fin d’une activité qui précède sa date de début.</p>
<p>Aucune donnée correspondant à l’année visionnée dans un graphique circulaire ou un tableau (p. ex. : lorsqu’un graphique circulaire présente des données pour 2016, tandis que les données publiées ne concernent que 2015).</p>
<p>Une mise à jour du profil dans le Registre n’apparaîtra qu’après l’importation complète hebdomadaire (chaque fin de semaine), car le d-portal n’actualise chaque nuit que les données qui ont été modifiées dans le fichier XML.</p>

#^faq_technical_q1_text
Quels types de transactions sont utilisés ou non dans d-portal?

#^faq_technical_a1_text
<p>On utilise « Dépense » ou « Décaissement » pour les données ou les colonnes relatives aux dépenses. S’il ne s’agit pas d’une dépense ou d’un décaissement, mais qu’il existe des données sur les « engagements », ces dernières apparaîtront.</p>
<p>On utilise « Budgets » du fichier Activités pour les données relatives au budget pour les données ou colonnes pertinentes (p.ex. : tableaux dans le d-portal). S’il n’y a aucune donnée budgétaire dans le fichier Activités, « décaissements prévus » sera utilisé s’il y en a.</p>

#^faq_technical_q2_text
Comment et où met-on de l’ordre dans les données avant de les afficher dans le d-portal?

#^faq_technical_a2_text
<ul>
	<li>Si la date d’une transaction n’est pas fournie, le d-portal utilise la date relative à la valeur de la transaction pour établir la date d’une transaction (et, par conséquent, lui assigner une année spécifique).</li>
	<li>Seuls les pays codés selon <a href="https://en.wikipedia.org/wiki/ISO_3166-2">ISO3166 Alpha-2</a> sont affichés.</li>
	<li>Les éditeurs secondaires – signalés comme des sources secondaires dans le Registre de l’IITA –  sont éliminés des tableaux afin d’éviter les répétitions (p. ex.le Bureau de coordination des affaires humanitaires [OCHA] des Nations Unies qui republie des données d’organisations déclarantes qui se trouvent dans les mêmes tableaux. Voici la <a href="https://github.com/devinit/D-Portal/issues/156">liste des éditeurs secondaires</a> exclus dans ces cas.)</li>
	<li>Les organisations qui n’ont aucun ensemble de données dans le Registre de l’IITA ne sont pas affichées.</li>
	<li>S’il n’existe aucune donnée relative aux dépenses ou aux décaissements, mais qu’il existe des données sur les engagements, ces dernières seront affichées.</li>
	<li>Afin d’établir une correspondance avec les données du SNPC dans les tableaux, les ministères sont regroupés et présentés sous le nom du pays. Les détails relatifs aux ministères sont consignés dans <a href="https://docs.google.com/spreadsheets/d/1jpXHDNmJ1WPdrkidEle0Ig13zLlXw4eV6WkbSy6kWk4/edit#gid=2">ctrack</a>.</li>
	<li><i>On examine actuellement la possibilité de n’utiliser que les données du niveau hiérarchique le plus bas, s’il existe des données à différents niveaux de hiérarchie, afin d’éviter les répétitions.</i></li>
</ul>

#^faq_technical_q3_text
Comment assigne-t-on les données à une année spécifique?

#^faq_technical_a3_text
En utilisant la date relative à la valeur transactionnelle ou la date de la transaction (p. ex. : dans les colonnes des dépenses des tableaux).
<p>En utilisant la date de fin des données budgétaires (p. ex. : dans les colonnes relatives au budget des tableaux).</p>

#^faq_technical_q4_text
Que doit-on considérer comme un projet « manquant »?

#^faq_technical_a4_text
Tout ce qui est délimité dans le temps par des dates incohérentes (p. ex. : une date de début ultérieure à la date de fin).
<p>Les projets dont l’échéancier comporte des dates incohérentes qui ne permettent pas de classer le projet dans l’une des trois catégories « en cours », « prévu » et « terminé ».</p>

#^faq_technical_q5_text
Comment calcule­t­on les taux de change?

#^faq_technical_a5_text
<ul>
	<li>En dollars américains par défaut.
	<p>Pour le moment (veuillez noter que cela pourrait bientôt changer), la conversion au dollar américain est fondée sur le taux de change à la date précisée dans l’attribut date-valeur. Par conséquent, lorsqu’un montant est affiché en dollar américain, celui-ci correspond très exactement à cette date.</p></li>

	<li>Autres devises
	<p>Pour ce qui est des autres devises, la valeur du dollar américain est simplement convertie en utilisant le taux de change du jour (veuillez noter que cela pourrait bientôt changer).</p></li>

	<li>À titre d’information budgétaire pour le futur?
	<p>On utilise les taux de change les plus récents (<a href="http://www.imf.org/external/np/fin/data/rms_five.aspx">Fonds monétaire international</a>) chaque nuit et procède à la réimportation complète une fois par semaine, de sorte à ce que les budgets pour l’avenir soient fondés sur les taux de change en cours et soient mis à jour hebdomadairement.</p></li>
</ul>

#^faq_technical_q6_text
Quels sont les critères pour que les données de mon organisation apparaissent dans les cartes, tableaux ou graphiques des pays bénéficiaires?

#^faq_technical_a6_text
Pour ce qui est des graphiques, il faut une transaction dont la date correspond à l’année appropriée (la dernière année complète, comme 2014 dans le cas présent) et ayant un code de pays bénéficiaire valide ou un code de secteur valide, selon le type de graphique.
<p>Il en va de même pour les tableaux, il faut une date de transaction se rapportant à la colonne de l’année appropriée dans le tableau.</p>
<p>En ce qui a trait aux cartes, il faut que les transactions (peu importe l’année) aient une étiquette d’emplacement et un code de pays bénéficiaire valides (pour qu’ils soient valides, ils doivent apparaître dans les listes de codes appropriées de l’IITA).</p>

#^faq_technical_q8_text
Qu’est-ce qui permet de classifier un projet comme étant « en cours », « terminé » ou « prévu »?

#^faq_technical_a8_text
Les dates de début et de fin d’une activité servent à déterminer les projets qui sont en cours, terminés ou prévus dans l’avenir.
<ul>
	<li>Les projets en cours sont ceux dont la date de début se situe dans le passé et la date de fin dans le futur.</li>
	<li>Les projets terminés sont ceux dont la date de fin se situe dans le passé.</li>
	<li>Les projets prévus sont ceux dont la date de début se situe dans le futur.</li>
</ul>

Comment produit-on les données cartographiques?

#^faq_technical_a9_text
On utilise des étiquettes d’emplacement dans les données afin de placer les projets sur les cartes, soit des données relatives à la latitude/longitude ou au point/positionnement, selon le choix de l’organisation déclarante.

#^faq_explore_q1_text
Qu’est-ce qu’EXPLORE?

#^faq_explore_a1_text
EXPLORE est composé de deux parties, soit d’une série de filtres et d’une boîte de texte libre.
<p><b>1. Les filtres</b></p>
<p>Il s’agit d’une série de filtres ayant trait à la <a href="http://iatistandard.org/">norme de l’IITA</a> permettant aux utilisateurs de choisir et de préciser les données de l’IITA qu’ils souhaitent explorer.</p>
<p><b>2. Texte libre</b></p>
<p>Il s’agit d’une boîte de texte libre permettant aux utilisateurs de rechercher des thèmes, des sujets et des activités précises publiées dans le cadre de l’IITA.</p>

#^faq_explore_q2_text
En quoi consiste EXPLORE?

#^faq_explore_a2_text
La fonction EXPLORE se trouve à la page d’accueil du <a href="http://d-portal.org">d-portal.org</a> et aide les utilisateurs à trouver des activités publiées selon la norme de l’IITA au moyen de mots-clés. À l’origine, EXPLORE était conçu pour aider les personnes à trouver des renseignements sur des crises humanitaires précises.

#^faq_explore_q3_text
Comment puis-je utiliser les filtres?

#^faq_explore_a3_text
On peut sélectionner plusieurs filtres parmi les listes déroulantes. À mesure que vous créerez votre requête en utilisant les filtres, le nombre d’activités consultables s’affichera sous les filtres selon vos choix.
<p>Vous pourrez ensuite cliquer sur « Explore Results » (explorer les résultats) afin d’en savoir davantage sur ces activités.</p>

#^faq_explore_q4_text
Quelles sont les données des listes déroulantes des filtres?

#^faq_explore_a4_text
<b>Pays bénéficiaires</b> et <b>organisations déclarantes</b>
<p>Il s’agit des organismes qui ont publié selon la norme de l’IITA, ainsi que les pays qui ont reçu des fonds, tels que publiés dans les données de l’IITA.</p>
<p><b>Groupe sectoriel et secteur</b></p>
<p>Il s’agit des codes numériques et des noms servant à décrire les secteurs au sein du SNPC du CAD de l’OCDE, utilisés dans le cadre de la norme de l’IITA.</p>
<p><b>Statut de l’activité</b></p>
<p>Il s’agit des codes numériques et des noms dans le champ relatif au statut de l’activité de l’IITA.</p>
<p><b>Périodes</b></p>
<p>De 1960 à aujourd’hui + 2.</p>

#^faq_explore_q5_text
Comment puis-je utiliser et préciser la recherche dans le texte libre? 

#^faq_explore_a5_text
Il suffit d’écrire un ou plusieurs mots-clés et d’appuyer sur « Entrée » afin de voir combien d’activités ont été trouvées.
<p>Cliquez sur « <b>Explore Results</b> » (explorer les résultats) pour voir des listes, des graphiques et des cartes permettant un examen approfondi des données.</p>
<p>Affinez votre recherche en ajoutant plusieurs mots. Par exemple, « <i>Ouragan Matthew</i> » générera plus de résultats précis (mais moins en nombre) que seulement « <i>Ouragan</i> ».</p>

#^faq_explore_q6_text
Comment apparaissent les résultats de la recherche avec EXPLORE?

#^faq_explore_a6_text
En cliquant sur « <b>Explore Results</b> » (explorer les résultats), les résultats apparaissent dans le format habituel du d-portal (sous forme de statistiques et de graphiques qu’il est possible d’explorer en cliquant dans les tableaux).

#^faq_explore_q7_text
Lorsqu’on effectue une recherche dans un texte libre, quels champs de l’IITA sont concernés par la recherche?

#^faq_explore_a7_text
Les champs <a href="http://iatistandard.org/202/activity-standard/iati-activities/iati-activity/title/">Titre</a> et <a href="http://iatistandard.org/202/activity-standard/iati-activities/iati-activity/description/">Description</a> de toutes les activités publiées de l’IITA font l’objet de la recherche.

#^faq_explore_q8_text
Est-il possible d’effectuer une recherche avec des noms au pluriel dans les textes libres?

#^faq_explore_a8_text
L’outil EXPLORE est adapté aux noms au pluriel en anglais. Par exemple, en écrivant « <i>réfugié</i> », « <i>réfugiés</i> » sera compris dans les résultats.

#^faq_generator_q1_text
De quoi s’agit-il?

#^faq_generator_a1_text
Le <a href="http://d-portal.org/ctrack.html#view=generator">Generator</a> (générateur) est un outil servant à aider les utilisateurs à reproduire les données du d-portal dans d’autres pages Web ou blogues.

#^faq_generator_q2_text
Pourquoi existe-t-il?

#^faq_generator_a2_text
Il a été créé afin d’aider les utilisateurs qui veulent afficher leurs propres données, ou les données qui les intéressent, dans leur propre site Web ou blogue.

#^faq_generator_q3_text
Comment dois-je l’utiliser?

#^faq_generator_a3_text
Il suffit de sélectionner des options dans les listes déroulantes et coller-copier le code fourni dans le HTML de la page Web.
<p>Pour obtenir de plus amples renseignements, consultez la page <a href="http://www.d-portal.org/news/2014-11-30-Interactive-IATI-data-d-portal-generator.html">http://www.d-portal.org/news/2014-11-30-Interactive-IATI-data-d-portal-generator.html</a></p>

#^faq_generator_q4_text
À quelle fréquence le contenu généré dans ma page Web sera-t-il mis à jour?

#^faq_generator_a4_text
Il s’agit d’un outil dynamique, le contenu s’actualisera donc automatiquement pendant la nuit. Si vous modifiez vos données, les changements apportés seront reflétés. Ainsi, lorsque votre site ou billet de blogue contient des informations tirées de Generator, celles-ci changeront en conséquence.

#^faq_generator_q5_text
Quel est le degré de fiabilité de cet outil?

#^faq_generator_a5_text
Les données intégrées dans votre site Web ou billet de blogue seront affichées aussi longtemps que le d-portal sera opérationnel (à savoir, de façon permanente).
<p>Le seul problème qui pourrait survenir est que votre fournisseur de site Web refuse que vous incorporiez des widgets externes à vos billets de blogue, mais si vous pouvez intégrer une vidéo YouTube à votre blogue, vous pouvez intégrer du contenu de Generator dans votre billet.</p>

#^faq_generator_q6_text
À l’aide! J’essaie de créer le widget dans le site Web interne de mon entreprise, mais ça ne fonctionne pas.

#^faq_generator_a6_text
Les accès HTTP protégé et non protégé sont permis et soutenus par d-portal. Les utilisateurs travaillant à partir de domaines protégés devraient pouvoir utiliser le contenu du Generator.
<p>Vous pouvez aussi utiliser <a href="https://d-portal.org/ctrack.html#view=generator">https://d-portal.org/ctrack.html#view=generator</a>.</p>
<p>Si vous n’êtes pas certain d’être visé, vérifiez si l’adresse du site Web de votre entreprise commence par http<b>s</b>:// plutôt que http://.</p>
